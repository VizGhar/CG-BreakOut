Dimensions of all game components

Bouncing [more descriptive bullets]:

  • Angle 0 points straight up; +angle is angle degrees clockwise, -angle is angle degrees counterclockwise
  • Of walls - standard
  • Of bricks - standard, if two adjacent bricks are hit at the same time both are removed
  • Of paddle - paddle is composed of 8 sections each adjusts ball angle by given amount of degrees (see image below; TODO: better explain)
  • See Expert rules to understand ball trajectory
Exceptions
  • Ball angle was >= 10 and now is between -10;10 = adjust to -10
  • Ball angle was <= -10 and now is between -10;10 = adjust to 10
  • Ball angle is < -70 = adjust to -70
  • Ball angle is > 70 = adjust to 70

8 sections of paddle (each has length of 8 pixels)
Victory Conditions
  • Destroy all bricks within a test-dependent turn limit.
Lose Conditions
  • Number of turns exceeds test's turn limit.
  • The ball leaves the screen.
  • Incorrect output format and/or value.

  Expert rules

Trajectory computation:

  • Take top-left corner of ball as start point
  • Compute target point at the edge of the board based on start and ballAngle (target is rounded to closest int)
  • Draw Bresenham line from start to target
  • If projection of ball hits obstacle - take previous point on Bresenham line, place the ball there and compute new ballAngle
  • If ball hits the brick from top or bottom side newBallAngle = 180 - ballAngle; -ballAngle otherwise

  Game Input

Input

Every turn you will receive current configuration in following format:

Line 1: remainingTurns or bounces of the paddle

Line 2: paddleX x-coordinate of paddle (paddleY coordinate is always 448 )

Line 3: space separated ballX and ballAngle (ballY coordinate is always 432; angle0pointing up)

Line 4: number n representing amount of blocks to destroy

Following n lines: block description. For example: 2 64 160 (blockHp, blockX, blockY)

Output

1 Line: required paddleX at the end of this turn (left side of paddle)

Constraints
0 ≤ paddleX ≤ 576
0 ≤ blockX ≤ 576; blockX is divisible by 64
0 ≤ blockY ≤ 256; blockY is divisible by 32
10 ≤ abs(ballAngle) ≤ 70
ballAngle is always divisible by 10
remainingTurns ≤ 200
Allotted response time to output is ≤ 2sin first turn and 100msin remaining turns
