Let's solve BreakOut! Legendary game made by Atari...

  The Goal

Destroy all blocks in all testcases in given amount of turns... 1 turn = 1 bounce of paddle.

  Rules

Blah blah:

  • Screen - 640×480
  • Coordinates [0,0] - top-left corner
  • Blocks - 80 possible placements
  • Block size - 64×32
  • Ball size - ball is for simplicity square with dimensions - 16×16 this will probably change to 17x17 so center of ball can be clearly described
  • Ball angle - always between 10,70 or -70,-10 degrees (including)
  • Paddle size - 64×16
  • Paddle Y position - 448 (screen height -32)

Dimensions of all game components

Bouncing:

  • Angle 0 points straight up, +20 is 20 degrees clockwise, -30 is 30 degrees counterclockwise
  • Of walls - standard
  • Of blocks - standard, if two blocks are hit at the same time both are removed
  • Of paddle - paddle is composed of 8 sections each changes ball angle by given amount of degrees I'd like to keep it this way, but maybe more granular changes will be necessary
  • After every bounce x and y coordinate of a ball are rounded to closest int This will most probably change to Bresenham algorithm to prevent most rounding errors
Exceptions
  • Ball angle was >= 10 and now is between -10;10 = adjust to -10
  • Ball angle was <= -10 and now is between -10;10 = adjust to 10
  • Ball angle is < -70 = adjust to -70
  • Ball angle is > 70 = adjust to 70

8 sections of paddle (each has length of 8 pixels)
Victory Conditions
  • Destroy all blocks within a test-dependent turn limit.
Lose Conditions
  • Number of turns exceeds test's turn limit.
  • The ball exists screen.
  • Incorrect output format and/or value.

  Game Input

Input

Every turn you will receive current configuration in following format:

Line 1: remainingTurns or bounces of the paddle

Line 2: paddleX x-coordinate of paddle (paddleY coordinate is always 448 )

Line 3: space separated ballX and ballAngle (ballY coordinate is always 432; angle0pointing up)

Line 4: number n representing amount of blocks to destroy

Following n lines: block description. For example: 15 2 64 160 (blockId, blockHp, blockX, blockY)

Output

1 Line: required paddleX at the end of this turn

Constraints
0 ≤ paddleX ≤ 576
0 ≤ blockX ≤ 576; blockX is divisible by 64
0 ≤ blockY ≤ 256; blockY is divisible by 32
10 ≤ abs(ballAngle) ≤ 70
remainingTurns ≤ 200
Allotted response time to output is ≤ 2sin first turn and 100msin remaining turns
